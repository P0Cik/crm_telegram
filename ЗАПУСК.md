# Инструкция по запуску

Backend системы мониторинга автомобильных предложений (импорт из Кореи).
Стек: Django + DRF, PostgreSQL, Celery + Redis, Encar-сборщик, Telegram-авторизация.

Есть два способа запуска: **через Docker** (рекомендуется, поднимает весь стек) и
**локально через venv** (удобно для разработки backend).

---

## Вариант A. Docker (полный стек)

### 1. Требования
- Docker и Docker Compose (v2+).

### 2. Подготовка окружения
Скопируйте пример переменных окружения и при необходимости отредактируйте:
```bash
cp .env.example .env
```
Минимум, что стоит указать в `.env`:
- `DJANGO_SECRET_KEY` — любая случайная строка;
- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather (нужен для уведомлений и
  проверки авторизации). Если бота пока нет — поставьте
  `TELEGRAM_AUTH_DEV_BYPASS=True`, чтобы входить без проверки подписи
  (**только для локальной разработки!**).

### 3. Запуск
```bash
docker compose up -d --build
```
Backend-контейнер при старте сам применит миграции и заполнит справочники
(`migrate` + `seed_brands`).

### 4. Что поднимется
| Сервис | Адрес | Назначение |
|---|---|---|
| Gateway (Nginx) | http://localhost:8080 | фронтенд + проксирование API |
| Backend API | http://localhost:8080/api/ | REST API |
| Swagger UI | http://localhost:8080/api/docs/ | документация API |
| Django Admin | http://localhost:8080/admin/ | админка менеджера |
| PostgreSQL | localhost:5432 | база данных |
| Redis | localhost:6379 | брокер Celery |

Также работают контейнеры `celery_worker`, `celery_beat` (периодический синк) и
`telegram_bot`.

### 5. Первичное наполнение данными
Создать суперпользователя (для входа в админку):
```bash
docker compose exec backend python manage.py createsuperuser
```
Загрузить демо-автомобили из примеров Encar (без обращения к сети):
```bash
docker compose exec backend python manage.py import_encar_fixtures
```
Запустить реальный сбор по профилю (профили видны в админке → «Профили сбора»;
там же кнопка-действие «Запустить синхронизацию»):
```bash
docker compose exec backend python manage.py shell -c "from cars.tasks import run_import_profile; run_import_profile(1)"
```
sync_catalog
### 6. Управление
```bash
docker compose ps                 # статус
docker compose logs -f backend    # логи backend
docker compose logs -f celery_worker
docker compose down               # остановить
docker compose up -d --build      # пересобрать после изменений
```

## Вариант B. Локально через venv (backend для разработки)

Подходит для быстрой разработки и тестов backend. Использует SQLite (без Postgres).

### 1. Требования
- Python 3.11–3.13 рекомендуется. На 3.14 часть бинарных пакетов (`aiogram`,
  `aiohttp`, `pydantic`) не имеет готовых сборок — для локального backend они не
  нужны (бот запускается отдельно в Docker), ставим облегчённый набор.

### 2. Виртуальная среда и зависимости
Из корня проекта (`crm_telegram/`):
```bash
python -m venv .venv
# Windows:
.venv\Scripts\python.exe -m pip install --upgrade pip
.venv\Scripts\python.exe -m pip install Django djangorestframework djangorestframework-simplejwt \
    django-cors-headers django-filter drf-spectacular httpx celery redis python-dotenv python-dateutil
```
> Для полного набора (с ботом) на Python 3.11:
> `.venv\Scripts\python.exe -m pip install -r crm_core/requirements.txt`

### 3. Миграции, справочники, демо-данные
Все локальные команды запускаются с `USE_SQLITE=1`:
```bash
cd crm_core
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py migrate
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py seed_brands
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py import_encar_fixtures
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py createsuperuser
```
> На Windows PowerShell: `$env:USE_SQLITE="1"; ..\.venv\Scripts\python.exe manage.py migrate`

### 4. Запуск dev-сервера
```bash
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py runserver
```
- API: http://localhost:8000/api/
- Swagger: http://localhost:8000/api/docs/
- Admin: http://localhost:8000/admin/

### 5. Тесты
```bash
USE_SQLITE=1 ../.venv/Scripts/python.exe manage.py test cars
```

## Основные эндпоинты API

| Метод | Путь | Описание |
|---|---|---|
| POST | `/api/auth/telegram/` | авторизация по `initData` → JWT |
| GET | `/api/cars/` | каталог (фильтры: brand, model, year_min/max, price_min/max, mileage_min/max, fuel_type, color…) |
| GET | `/api/cars/{id}/` | карточка авто (цена won/rub, фото, опции) |
| GET | `/api/cars/filters/` | доступные значения фильтров для UI |
| GET/POST/PATCH/DELETE | `/api/search-requests/` | подписки на фильтры |
| GET/POST | `/api/orders/` | заявки (бронь), создаёт клиент |
| POST | `/api/orders/{id}/update_status/` | смена статуса (менеджер) |
| GET | `/api/orders/{id}/history/` | таймлайн статусов с фото |
| GET/POST | `/api/search-profiles/` | профили сбора Encar (менеджер) |
| POST | `/api/search-profiles/{id}/run/` | запустить синхронизацию профиля |

## Как работает сбор и уведомления
1. Менеджер создаёт **Профиль сбора** (марка + группа модели) в админке.
2. Celery-задача `sync_encar_profile` тянет список Encar, нормализует и сохраняет
   авто (дедуп по `source+external_id`), деактивирует пропавшие, дозагружает детали.
3. Для новых авто запускается матчинг подписок: подходящие — **сразу
   отправляются пользователю в Telegram** (без хранения в БД, дедуп по watermark).
4. `celery_beat` запускает синхронизацию всех активных профилей каждые 30 минут.

## Переменные окружения (ключевые)
| Переменная | По умолчанию | Назначение |
|---|---|---|
| `DJANGO_SECRET_KEY` | dev-ключ | секрет Django |
| `DEBUG` | True | режим отладки |
| `TELEGRAM_BOT_TOKEN` | — | токен бота (уведомления + проверка авторизации) |
| `TELEGRAM_AUTH_DEV_BYPASS` | False | вход без проверки подписи (только dev!) |
| `KRW_RUB_RATE` | 0.065 | курс RUB за 1 KRW (для конвертации цены) |
| `ENCAR_IMAGE_BASE` | https://ci.encar.com | CDN для фото авто |
| `ENCAR_REQUEST_DELAY` | 1.0 | пауза между запросами к Encar (сек) |
| `USE_SQLITE` | — | `1` = использовать SQLite вместо Postgres (локально) |

## Troubleshooting
- **401 на `/api/...`** — нужен JWT. Сначала `POST /api/auth/telegram/` с `initData`.
  Для локальной проверки без бота поставьте `TELEGRAM_AUTH_DEV_BYPASS=True`.
- **Пустой каталог** — выполните `import_encar_fixtures` (демо) или запустите
  синхронизацию профиля.
- **Уведомления не приходят** — проверьте `TELEGRAM_BOT_TOKEN` и что у пользователя
  заполнен `telegram_id` (заполняется при входе через Telegram).
- **Локально не ставится `aiogram`/`pydantic` на Python 3.14** — это нормально, для
  backend они не нужны; ставьте облегчённый набор (см. Вариант B) или используйте Docker.
